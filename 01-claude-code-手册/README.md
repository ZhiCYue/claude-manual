## 学习路径建议（4 周）

```
第 1 周  00–02 章：日常对话式开发。目标：养成 /clear、@文件、! 命令、Esc 中断的肌肉记忆
第 2 周  03–04 章：为你的常用项目写好 CLAUDE.md；复杂任务一律先进 Plan Mode
第 3 周  05–06 章：沉淀 2 个自定义命令或 Skill；配置 2–3 个 Hook
第 4 周  07–09 章：接入 1 个 MCP；创建 1 个子代理；写 1 个用 claude -p 的脚本
之后     10–11 章：综合运用 + 大项目降本。标志性转变——你开始「为 AI 设计环境」而不只是和它聊天
```

## 全书一条主线

所有高级功能其实都在回答同一个问题：**如何在「模型每次会话都失忆、上下文窗口有限、行为有随机性」这三个约束下，获得可靠、高效的工程产出？**

- 失忆 → CLAUDE.md、Skills、文档接力（03、05、10、11 章）
- 窗口有限 → /clear、/compact、子代理、降本工程（02、08、11 章）
- 有随机性 → 权限、Hooks、测试驱动、计划先行（04、06、10 章）

带着这条主线读，每个功能的设计意图都会一目了然。
