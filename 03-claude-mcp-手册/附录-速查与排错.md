## A. FastMCP 装饰器速查（Python）

```python
from mcp.server.fastmcp import FastMCP
mcp = FastMCP("server-name")

# 工具：模型主动调用的函数
@mcp.tool()
def my_tool(arg: str) -> str:
    """工具说明——模型靠它决定何时调用。"""
    return "结果"

# 工具可加注解
@mcp.tool(annotations={"readOnlyHint": True})
def read_only_tool() -> str: ...

# 资源：固定 URI 的只读数据
@mcp.resource("config://settings")
def my_resource() -> str:
    return "数据"

# 资源模板：带参数的 URI
@mcp.resource("users://{user_id}/profile")
def user_profile(user_id: str) -> str:
    return f"用户 {user_id}"

# 提示词：用户触发的模板
@mcp.prompt()
def my_prompt(topic: str) -> str:
    return f"请帮我处理 {topic}"

# 启动
if __name__ == "__main__":
    mcp.run()                              # stdio（默认）
    # mcp.run(transport="streamable-http") # 远程 HTTP
```

## B. 三原语速查

| 原语 | 装饰器 | 谁控制 | 协议方法 | 类比 |
|---|---|---|---|---|
| Tools | `@mcp.tool()` | 模型 | `tools/list`、`tools/call` | 函数调用 |
| Resources | `@mcp.resource(uri)` | 应用 | `resources/list`、`resources/read` | GET / 文件 |
| Prompts | `@mcp.prompt()` | 用户 | `prompts/list`、`prompts/get` | 斜杠命令 |

**选型口诀**：模型该主动调 → Tool；只读上下文 → Resource；用户主动触发的模板 → Prompt。

## D. 接入宿主速查

```bash
# Claude Code —— 本地 stdio
claude mcp add 名字 -- python /绝对路径/server.py
# Claude Code —— 远程 HTTP
claude mcp add --transport http 名字 https://host/mcp
# Claude Code —— 作用域
claude mcp add --scope project 名字 ...   # 团队共享（生成 .mcp.json）
claude mcp list / get 名字 / remove 名字
```

Claude 桌面端 `claude_desktop_config.json`：

```json
{
  "mcpServers": {
    "名字": {
      "command": "python",
      "args": ["/绝对路径/server.py"],
      "env": { "API_KEY": "..." }
    }
  }
}
```

位置：macOS `~/Library/Application Support/Claude/claude_desktop_config.json`，Windows `%AppData%\Claude\claude_desktop_config.json`。改完重启。


## F. 架构速查

```
宿主（AI 应用）
  └── 客户端 1 ──连接── 服务器 A（你写的）
  └── 客户端 2 ──连接── 服务器 B

两层：
  数据层  = JSON-RPC 2.0（消息内容、三原语、生命周期）
  传输层  = stdio（本地）或 Streamable HTTP（远程）

生命周期：initialize 握手 → 能力协商 → notifications/initialized → 正常通信
```

## G. 常见问题排错（FAQ）

**Q1：服务器连不上 Claude Code/桌面端？**
先在终端手动跑配置里的启动命令，看能否正常启动、有无报错。检查：用了绝对路径吗？`command` 是完整路径吗（`which python`/`which uv`）？

**Q2：连上了但工具不显示？**
用 Inspector（`uv run mcp dev server.py`）看 `tools/list` 是否返回你的工具。检查装饰器是否正确、服务器是否真的注册了工具。

**Q3：连接莫名其妙断开？**
十有八九是 stdout 被污染——有 `print()` 或别的输出混进了 JSON-RPC 通道。所有调试输出改走 stderr 或日志文件。这是 stdio 服务器最常见的坑。

**Q4：工具该做成工具还是资源？**
要模型主动做动作（可能有副作用）→ 工具；只读上下文供参考 → 资源。见第 3.7 节判断清单。

**Q5：本地服务器能在 Claude API 里用吗？**
API 直连只支持远程 HTTP 服务器（API 跑在 Anthropic 那边，连不到你本地进程）。本地服务器用 Agent SDK，或 `anthropic[mcp]` 的 MCP 转换助手（第 6.4 节）。

**Q6：怎么把服务器分享给团队？**
stdio 服务器：用 `claude mcp add --scope project` 生成 `.mcp.json` 提交进 git。远程服务器：部署成 HTTP 服务，团队连同一个 URL。把服务器代码、配置、README 一起进仓库。

**Q7：远程服务器怎么鉴权？**
Streamable HTTP 支持 Bearer token、API key、OAuth（推荐）。加鉴权中间件验证请求头的 token。务必同时做限流和输入验证。见第 5.6、7.5 节。

**Q8：工具返回的外部内容安全吗？**
警惕提示词注入——网页、用户数据、第三方 API 响应可能含恶意指令。别让返回内容被当指令执行，危险操作不依赖单一工具调用，结合宿主的权限机制。见第 7.5 节。

**Q9：异步还是同步工具？**
涉及网络/数据库/文件 IO 用 `async def`，CPU 密集用同步 `def`。FastMCP 两者都支持，可混用。

**Q10：MCP 服务器和 Agent SDK 的进程内服务器（create_sdk_mcp_server）有什么区别？**
本册写的是独立 MCP 服务器（单独进程，任何宿主可连）。Agent SDK 的 `create_sdk_mcp_server`（第二册 8.5）是进程内的、只给那个 Agent 用的轻量版。要复用、要给多个宿主用 → 写独立服务器；只给一个 Agent 程序用 → 进程内更省事。

## I. 全册回顾：一个协议，三种原语

```
MCP = 用 JSON-RPC 让 AI 应用（宿主）和外部系统（服务器）交换上下文的标准协议

你写服务器，暴露三种原语：
  Tools     模型控制   能调用的函数        @mcp.tool()
  Resources 应用控制   只读数据源          @mcp.resource(uri)
  Prompts   用户控制   指令模板            @mcp.prompt()

写一次，处处可用：同一个服务器接 Claude Code / 桌面端 / API / Agent SDK / 其他 IDE
```

**安全是底线**：你的服务器跑在用户机器上、能访问用户数据、AI 自动调用它——当成公开 API 做安全设计。
