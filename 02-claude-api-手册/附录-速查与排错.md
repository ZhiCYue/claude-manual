## A. 模型 ID 速查

> 只用精确字符串，不要加日期后缀（`claude-haiku-4-5-20251001` 是例外，它是 Haiku 的完整 ID）。

| 友好名 | 模型 ID | 上下文 | 最大输出 | 输入/输出 $/百万 |
|---|---|---|---|---|
| Claude Fable 5 | `claude-fable-5` | 1M | 128K | $10 / $50 |
| Claude Opus 4.8 | `claude-opus-4-8` | 1M | 128K | $5 / $25 |
| Claude Opus 4.7 | `claude-opus-4-7` | 1M | 128K | $5 / $25 |
| Claude Opus 4.6 | `claude-opus-4-6` | 1M | 128K | $5 / $25 |
| Claude Sonnet 4.6 | `claude-sonnet-4-6` | 1M | 64K | $3 / $15 |
| Claude Haiku 4.5 | `claude-haiku-4-5` | 200K | 64K | $1 / $5 |

**用户说什么 → 用哪个**：「最强/most powerful」→ `claude-fable-5`；「opus」→ `claude-opus-4-8`；「sonnet/平衡」→ `claude-sonnet-4-6`；「haiku/快/便宜」→ `claude-haiku-4-5`。

**实时查能力**：用 `client.models.retrieve("claude-opus-4-8")` 拿 `max_input_tokens`、`max_tokens`、`capabilities`（vision/thinking/effort 支持等），或 `client.models.list()` 遍历。

## B. stop_reason 速查

| 值 | 含义 | 处理 |
|---|---|---|
| `end_turn` | 自然结束 | 处理结果 |
| `max_tokens` | 撞输出上限 | 调大 `max_tokens` 或流式 |
| `tool_use` | 想调工具 | 执行工具喂回（第 4 章） |
| `pause_turn` | 服务端工具暂停 | 原样发回继续（第 5 章） |
| `refusal` | 安全拒绝 | 检查 `stop_details`，别重试 |

## C. HTTP 错误码速查

| 码 | 类型 | 可重试 | 常见原因 |
|---|---|---|---|
| 400 | `invalid_request_error` | 否 | 请求格式/参数错误 |
| 401 | `authentication_error` | 否 | API key 无效或缺失 |
| 403 | `permission_error` | 否 | key 无权限 |
| 404 | `not_found_error` | 否 | 模型 ID 或端点错误 |
| 413 | `request_too_large` | 否 | 请求超大 |
| 429 | `rate_limit_error` | 是 | 请求过多（看 `retry-after` 头） |
| 500 | `api_error` | 是 | 服务端问题 |
| 529 | `overloaded_error` | 是 | 临时过载 |

429/5xx，SDK 默认指数退避自动重试 2 次（`max_retries` 可配）。

## F. 参数位置速查（容易放错的）

| 参数 | 正确位置 | 常见错误 |
|---|---|---|
| `effort` | `output_config={"effort": "high"}` | ❌ 放顶层 |
| `format` | `output_config={"format": {...}}` | ❌ 用废弃的 `output_format` |
| `thinking` | 顶层 `thinking={"type": "adaptive"}` | ❌ 用废弃的 `budget_tokens` |
| `cache_control` | 内容块上，或顶层自动 | — |
| 工具结果 | `user` 角色消息里 | ❌ 放 assistant |

## G. 避坑清单（FAQ）

**Q1：`response.content[0].text` 报错？**
content 是块列表，第一个块未必是文本（可能是 thinking 块）。先判断 `.type`：`next(b.text for b in response.content if b.type == "text")`。

**Q2：模型"不记得"上一轮说的话？**
API 无状态。每次请求必须带完整 `messages` 历史，包括之前的 assistant 回复。见第 1 章 1.4。

**Q3：工具循环报错，模型不知道自己调了什么？**
每轮要把完整 `response.content`（含 tool_use 块）追加进 `messages`，且每个 `tool_result` 的 `tool_use_id` 要匹配。见第 4 章 4.3。

**Q4：`budget_tokens` 报 400？**
Fable 5 / Opus 4.8 / 4.7 已移除它。用 `thinking={"type": "adaptive"}` + `output_config={"effort": ...}`。

**Q5：prompt caching 不命中（`cache_read_input_tokens` 总是 0）？**
有静默失效器——系统提示里的时间戳/UUID、未排序的 `json.dumps`、每请求变化的工具集。见第 6 章 6.6 审计表。

**Q6：Fable 5 请求莫名 400，payload 看着没问题？**
检查组织的数据保留配置——Fable 5 需要 30 天保留，ZDR 组织每个请求都会 400。

**Q7：长对话压缩后状态丢失？**
每轮要追加 `response.content`（完整内容块），不是只追加文本。压缩块必须保留。见第 7 章 7.3。

**Q8：非流式大输出请求超时/抛 ValueError？**
`max_tokens` 超过约 16K 要用流式。SDK 会拒绝它估算超过 10 分钟的非流式请求。见第 3 章。

**Q9：数 token 该用什么？**
`client.messages.count_tokens(model=..., messages=...)`，绝不用 tiktoken（OpenAI 的分词器，对 Claude 误差大）。

**Q10：Managed Agents 报错说 session 不接受 model/system/tools？**
那些字段在 agent 上，不在 session 上。先 `agents.create()`，session 只引用 agent ID。见第 9 章 9.2。

## I. 三层架构选型（再回顾）

```
单次调用（分类/摘要/提取/问答）           → Messages API（第 1–3 章）
多步流水线，代码控制流程                  → Messages API + Tool Use（第 4 章）
自定义 agent，自己的工具，自己托管计算     → 自建循环（第 4 章）或 Agent SDK（第 8 章）
服务端有状态 agent，云端容器，文件挂载     → Managed Agents（第 9 章）
```

**起点永远是最简单的层级。** 单次调用和工作流能覆盖大多数需求；只有任务确实需要开放式、模型驱动的探索时才上 agent。
