# 第 11 章 · 大项目降本与上下文工程

第 10 章解决"怎么干得好",这一章解决一个更现实的问题：**面对一个体量庞大的历史项目，每次让 Claude「从零理解再开发」都会触发大量盲目扫描，token 消耗惊人。** 本章把这种浪费的根因和对策系统化。

## 11.1 根因：扫描成本为什么被重复支付

每次让 Claude「从零理解项目」，它都会做大量 `grep` / `read` 全量探索。问题不在探索本身，而在于：**这些探索结果用完即弃**——会话一结束（或一次 `/clear`），下一轮又得从头扫一遍。

所以优化的本质就两件事：

1. **沉淀理解**——把项目结构、约定一次性写下来，长期复用（对应失忆约束⓪）。
2. **精确定位**——用索引和工具让模型直达相关代码，跳过盲目扫描（对应窗口有限约束⓪）。

下面六节就是围绕这两件事展开的。

## 11.2 分层 CLAUDE.md：把「理解」固化成自动加载的资产

（用到：CLAUDE.md③）单个根 `CLAUDE.md` 对大项目往往不够，推荐分层：

```
project/
├── CLAUDE.md                 # 全局架构地图 + 各模块一句话定位
├── src/api/CLAUDE.md         # 进入该目录工作时才加载
├── src/services/CLAUDE.md
└── docs/ARCHITECTURE.md      # 模块边界、数据流、依赖关系
```

根文件只放「地图」，例如：

```markdown
# 项目架构
- 后端: src/api/ (FastAPI), 入口 src/api/main.py
- 核心业务逻辑: src/services/
- 数据模型: src/models/ (SQLAlchemy)
- 前端: web/ (React + Vite)

# 关键约定
- 所有 DB 操作走 repository 层，禁止在 service 里直接写 SQL
- 新增 API 必须在 src/api/routes/ 注册并加 pytest
- 错误码定义在 src/common/errors.py

# 常用命令
- 测试: pytest tests/ -x   启动: make dev
```

原理：根 `CLAUDE.md` 每次启动自动加载、**零探索成本**；子目录的 `CLAUDE.md` 按需加载，Claude 在哪个模块干活才装哪份，不会一次性把全库说明塞进窗口。可用 `/init` 生成初版再人工精简——**只写地图和约定，绝不把代码搬进去**（否则违背 3.x 的百行原则）。

## 11.3 探索交给子代理：把脏活隔离在另一个窗口

（用到：子代理⑧）这是本章最省 token 的一招。当确实需要搜索时，**派 `Explore` / `general-purpose` 子代理去搜**：

```
> 用 Explore 子代理找出鉴权逻辑的入口和核心实现位置，
  只告诉我文件:行号 和调用链，不要贴大段代码
```

- 子代理在**独立的上下文窗口**里做大量 grep / read；
- 它只把**结论**回传主对话（"鉴权在 `auth/middleware.py:42` 的 `verify_token`"）；
- 中间那一堆文件 dump **不会污染你的主上下文**。

「理解 → 定位」的脏活在子窗口完成，主窗口保持干净。项目越大，这个隔离的收益越明显——参见 10.7 的代码考古流程，它本质就是这一招的组合应用。

## 11.4 给坐标，别让它盲扫

养成给模型**明确坐标**的习惯，是每次都生效的省钱手段：

| 写法 | 模型行为 | 成本 |
|---|---|---|
| ❌「帮我改一下退款功能」 | 全库搜索定位 | 高 |
| ✅「改 `src/services/payment.py` 的退款逻辑」 | 直接读该文件 | 低 |

配合：
- 让它先 `git log --oneline -- <path>` 或精确 grep 符号，而不是读整个目录；
- 用 `git blame` 取演进信息，比读代码猜更省。

## 11.5 用 ignore 缩小扫描面

从根上减少可被扫描的文件，盲扫时的代价也随之降低。配置 `.claudeignore`（或 settings 的相应排除），把这些挡在外面：

```
node_modules/
dist/
build/
*.min.js
**/generated/
data/*.csv        # 大数据/二进制文件
```

原理：探索类工具不会去碰被 ignore 的路径，等于直接削掉了无效扫描面。

## 11.6 会话与记忆策略

- **一会话一模块**：别在超长会话里反复横跨整个项目——上下文越长越贵、越易丢失（窗口经济学⓪）。模块切换时 `/clear`。
- **持久记忆**：把「非显而易见的约定/坑」写进项目记忆，跨会话复用（③），而不是每次重新解释。
- **超大改造用文档接力**：见 10.4——计划和进度放磁盘，每个会话只装当前阶段。

## 11.7 优先级速查表

| 优先级 | 做法 | 收益 | 章节 |
|---|---|---|---|
| ⭐⭐⭐ | 写好分层 CLAUDE.md | 一劳永逸，自动加载 | 11.2 / 3 |
| ⭐⭐⭐ | 探索任务交给 Explore 子代理 | 隔离脏上下文 | 11.3 / 8 |
| ⭐⭐ | 给明确文件坐标，不让它盲扫 | 每次都省 | 11.4 |
| ⭐⭐ | 配置 ignore 排除无关目录 | 缩小扫描面 | 11.5 |
| ⭐ | 维护 ARCHITECTURE.md + 持久记忆 | 长期资产 | 11.2 / 6 |

## 11.8 落地清单（接手一个历史项目时）

1. [ ] 跑 `/init` 生成根 `CLAUDE.md`，精简成「架构地图 + 约定 + 常用命令」。
2. [ ] 给 3–5 个核心模块各补一份子目录 `CLAUDE.md`。
3. [ ] 配置 `.claudeignore`，排除依赖、产物、生成代码、大文件。
4. [ ] 写一份 `docs/ARCHITECTURE.md`，画清模块边界与数据流（让 Claude 用 mermaid 出图，见 10.7）。
5. [ ] 形成习惯：探索派子代理、开发给坐标、一会话一模块。
