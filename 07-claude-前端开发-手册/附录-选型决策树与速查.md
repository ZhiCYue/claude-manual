# 附录 · 选型决策树与速查

## A. 构建工具/框架选型决策树(2026)

```
全新项目?
├── 要 SSR/SEO/全栈        → Next.js(App Router,Turbopack)
├── 纯客户端 SPA           → Vite + React
└── 微前端                 → webpack + Module Federation

已有项目?
├── 已在 webpack           → 留着,除非慢到严重拖累才评估迁移 Vite
└── 已在 Vite/Next         → 继续

⚠️ 不要用 CRA(已停止维护)。AI 若建议 CRA,那是过时知识。
```

## B. 两大支柱速查

```
支柱一 好环境(让 AI 好干活)        支柱二 规格驱动(让 AI 不跑偏)
─────────────────────────         ──────────────────────────
· 选对构建工具(第1-2章)            · 写码前定 spec(第6章)
· 现代 React 模式(第3章)           · 审 spec(最便宜的纠错点)
· AI 友好的结构(第4章)             · AI 照 spec 实现
· 测试作地面真相(第4章)            · OpenSpec 自动化(第7章)
· 工具中立质量门(第5章)            · spec 工具中立,混合团队共享
```

## C. React 19 速查

| 特性 | 用途 | 场景 |
|---|---|---|
| Server Components | 服务端渲染、直接取数、零客户端 JS | Next.js(框架) |
| Server Actions | 客户端调服务端函数,自动缓存重验证 | Next.js |
| `useActionState` | 自动管 action 的 loading/成功/错误 | 表单 |
| `useFormStatus` | 读表单提交状态 | 表单 |
| `useOptimistic` | 乐观更新 | 配 Server Actions |
| `use()` | 读 Promise/Context,可条件调用 | 配 Suspense |
| React Compiler | 自动 memo,**别再手动 useMemo/useCallback** | 全局 |

状态三分法:本地→`useState`;服务端→TanStack Query(SPA)/ Server Components(框架);全局→Zustand/Redux Toolkit。**别把服务端数据塞全局 store。**

## D. webpack 5 速查(给用 webpack 的人)

```
转译     swc-loader / esbuild-loader（替代 babel-loader）
资源     asset/resource（webpack 5 内置,替代 file-loader）
输出     [contenthash] + clean: true
分割     动态 import() + React.lazy/Suspense（路由懒加载）
分包     splitChunks: { chunks: "all" }（拆 vendor）
分析     webpack-bundle-analyzer
微前端   Module Federation
```

## E. OpenSpec 命令速查

```bash
npm install -g @fission-ai/openspec    # 安装(需 Node 20.19+)
openspec init                          # 项目里初始化
openspec update                        # 更新各助手的指令/命令
openspec config profile                # 选更丰富的工作流

# 在 Claude Code 等助手里:
/opsx:propose "功能名"    # 提案:生成 proposal/specs/design/tasks → 审 spec
/opsx:apply               # 实现:照 tasks 逐项做
/opsx:archive             # 归档:更新正式 specs
```

目录:`openspec/changes/<功能>/{proposal.md, design.md, tasks.md, specs/}` + `archive/`。全进 git。

## F. SDD/OpenSpec 何时用

| 用 | 不用 |
|---|---|
| 大功能、新页面/模块 | 改文案/样式 |
| 需求模糊易理解偏 | 需求明确的小修 |
| 多人协作要对齐 | 个人探索性尝试 |
| 重构/迁移(难回滚) | 一行 bug 修复 |

口诀:**任务越大、越模糊、越多人、越难回滚,SDD 越值。**

## G. 反模式清单

| ❌ | ✅ | 章 |
|---|---|---|
| 新项目用 CRA/webpack | Vite/Next | 1 |

| spec 不审就 apply | propose 后认真审 | 7 |
| 每个任务都走完整 SDD | 按规模简化(小的直接做) | 8 |

## J. 全册一页纸

```
核心:真实前端(React)项目里怎么把 AI 辅助开发做好?
```
