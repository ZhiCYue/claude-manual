# 第 0 章 · 总览：三层架构与模型选择

这一章建立全册的心智模型，并给出你每次写代码都要查的两张表：模型 ID 表和选型决策表。

---

## 0.1 一个端点统治一切

Claude API 的核心只有一个端点：

```
POST https://api.anthropic.com/v1/messages
```

你能想到的几乎所有能力——多轮对话、工具调用、流式输出、结构化 JSON、图片理解、思考——**都是这一个端点的参数**，不是独立的 API。理解这一点能省掉很多困惑：你不是在学几十个 API，你是在学一个端点的几十个参数。

少数辅助端点围绕它服务：

| 端点 | 用途 | 章节 |
|---|---|---|
| `/v1/messages/batches` | 批处理，半价异步 | 7 |
| `/v1/files` | 上传文件，跨请求复用 | 7 |
| `/v1/messages/count_tokens` | 精确数 token | 附录 |
| `/v1/models` | 查询模型能力与上下文窗口 | 附录 |

## 0.2 三层架构：从简单到复杂

构建 LLM 应用的第一原则是：**用能满足需求的最简单层级**。不要一上来就建 Agent。

### 第一层 · 单次调用

输入 → 输出，一次往返。适合：分类、摘要、信息提取、问答、翻译、改写。

```python
import anthropic
client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-opus-4-8",
    max_tokens=1024,
    messages=[{"role": "user", "content": "把这段话总结成一句话：……"}],
)
print(response.content[0].text)
```

绝大多数"用 AI 处理文本"的需求都停在这一层。

### 第二层 · 工作流（Workflow）

多步骤，但**流程由你的代码控制**。模型在每一步做一件确定的事，你用普通的 `if`/`for`/函数调用把这些步骤串起来。典型：先让模型分类工单 → 再根据类别用不同 prompt 处理 → 最后汇总。

工作流通常配合 **Tool Use**（第 4 章）：你定义工具，模型在需要时请求调用，你执行后把结果喂回去。

### 第三层 · Agent（代理）

**模型自己决定下一步做什么**，自主探索直到完成目标。这正是第一册讲的 Agent Loop。三种实现路径：

| 路径 | 你负责什么 | 适合 |
|---|---|---|
| 自建循环（第 4 章） | 自己写 while 循环、自己执行工具 | 最大灵活性，要审批/日志/人在回路 |
| Agent SDK（第 8 章） | 用库，循环由 SDK 跑 | 想要 Claude Code 的引擎但宿主是你的程序 |
| Managed Agents（第 9 章） | 几乎什么都不用，Anthropic 托管 | 要云端容器、文件挂载、跨会话状态 |

### 该不该建 Agent？四问

建 Agent 前，四个问题都答"是"才值得：

1. **复杂度**：任务是否多步且难以提前完全规定？（"把设计文档变成 PR" vs "提取 PDF 标题"）
2. **价值**：结果是否值得更高的成本和延迟？
3. **可行性**：Claude 在这类任务上确实能干好吗？
4. **错误成本**：错了能否被发现和恢复？（有测试、有审查、能回滚）

任何一个答"否"，就退回更简单的层级。

## 0.3 模型目录（务必用精确 ID）

> ⚠️ **只用下表的精确字符串，不要拼接、不要加日期后缀。** 用 `claude-opus-4-8`，绝不要写 `claude-opus-4-8-20251114`。猜测 ID 会直接 404。

| 模型 | 模型 ID | 上下文 | 最大输出 | 输入 $/百万 | 输出 $/百万 |
|---|---|---|---|---|---|
| **Claude Fable 5** | `claude-fable-5` | 1M | 128K | $10 | $50 |
| **Claude Opus 4.8** | `claude-opus-4-8` | 1M | 128K | $5 | $25 |
| Claude Opus 4.7 | `claude-opus-4-7` | 1M | 128K | $5 | $25 |
| Claude Opus 4.6 | `claude-opus-4-6` | 1M | 128K | $5 | $25 |
| **Claude Sonnet 4.6** | `claude-sonnet-4-6` | 1M | 64K | $3 | $15 |
| **Claude Haiku 4.5** | `claude-haiku-4-5` | 200K | 64K | $1 | $5 |

### 选哪个？

- **`claude-opus-4-8`** —— 默认选它。最强的 Opus 级模型，高度自主，长程 agent 任务、知识工作、记忆任务都是当前最佳。这是绝大多数应用的正确起点。
- **`claude-fable-5`** —— Anthropic 最强的通用可用模型，专为最艰难的推理和长程 agent 工作设计。定价高于 Opus 级；只在用户明确要求或任务确实极难时用。它的 API 行为和 Opus 系列有差异（见下方）。
- **`claude-sonnet-4-6`** —— 速度与智能的最佳平衡。高并发生产负载、常规编码、批量处理用它，省一半成本。
- **`claude-haiku-4-5`** —— 最快最省。简单分类、海量小任务、对延迟敏感的场景。

> **本册代码示例统一用 `claude-opus-4-8`**，除非某个例子专门演示模型差异。你按需替换即可。

### Fable 5 的几个特殊之处（用到再看）

如果你用 `claude-fable-5`，注意它和 Opus 系列的 API 差异：

- **思考永远开启**：不要传 `thinking` 参数（或传 `{type: "adaptive"}`）。传 `{type: "disabled"}` 或 `{type: "enabled", budget_tokens: N}` 都会 400。用 `output_config.effort` 控制深度。
- **原始思维链永不返回**：响应里是常规 `thinking` 块，`display: "summarized"` 给摘要，默认 `"omitted"` 时 `thinking` 字段为空字符串。
- **可能返回 `refusal` 停止原因**：安全分类器可能拒绝请求（HTTP 200，`stop_reason: "refusal"`）。读 `content` 前先检查 `stop_reason`。
- **需要 30 天数据保留**：零数据保留（ZDR）的组织调用 Fable 5 会 400。

## 0.4 三条贯穿全册的硬规则

再强调一遍，这三条你会反复用到：

1. **模型 ID 用精确字符串**（0.3 节的表）。
2. **API 无状态**——每次请求带完整历史。"会话""记忆"都是你在客户端维护历史、每次重新发送实现的（第 1 章详解）。
3. **数 token 用 `count_tokens`**，不要用 tiktoken。

## 0.5 鉴权与环境准备

```bash
pip install anthropic              # Python SDK
# 或
npm install @anthropic-ai/sdk      # TypeScript SDK

export ANTHROPIC_API_KEY=sk-ant-...   # 从 console.anthropic.com 获取
```

```python
import anthropic
client = anthropic.Anthropic()     # 自动读 ANTHROPIC_API_KEY 环境变量
```

**永远不要把 API Key 硬编码进代码**——用环境变量或密钥管理服务。泄露的 key 等于泄露了你的账单。

企业可走 AWS Bedrock / Google Vertex AI / Microsoft Foundry 作为后端，模型 ID 和部分功能可用性会变（见附录）。
