# 新手上路：从零搭建你的第一个 AI 辅助前端项目

> **面向完全的新手。** 假设你的电脑是"干净"的——没装过任何开发工具。我们从装软件开始,一步步搭起一个前端项目骨架,接上 AI 编程助手,并教你之后怎么用 AI 辅助开发。
>
> 跟着做,你会得到:一个能在浏览器跑起来的前端项目 + 连好的 Claude Code(AI 助手)+ 一套你能重复的日常开发流程。预计 1–2 小时。

---

## 开始之前:几个名词(30 秒)

不用记,遇到了回来看:

- **终端 / 命令行**:一个让你输入文字命令来操作电脑的窗口。开发离不开它。
- **Node.js**:前端项目运行和构建需要的"引擎"。装了它才有 `node` 和 `npm` 命令。
- **npm**:Node 自带的"包管理器",用来下载别人写好的代码库(如 React)。
- **编辑器(VS Code)**:写代码的软件,比记事本强大得多。
- **Git**:记录代码改动历史的工具(像游戏存档),还能多人协作。
- **Claude Code**:跑在终端里的 AI 编程助手——本系列的主角。
- **脚手架**:一条命令帮你生成一个项目的初始骨架,省得手动建一堆文件。

---

# 阶段一 · 装软件(含 AI 工具)

按顺序装。每步都有"检查"——确认装好了再往下。

## 1. 先打开终端

- **macOS**:按 `Command + 空格`,输入 `Terminal`(终端),回车。
- **Windows**:开始菜单搜 `PowerShell`,打开它。(更好的选择是装 [Windows Terminal](https://aka.ms/terminal),但 PowerShell 也能用。)

> 之后所有"运行某命令"都是指:在这个终端窗口里输入命令、按回车。

**检查**:窗口里能输入文字、按回车有反应,就行。

## 2. 装 Node.js(前端的引擎)

去 [nodejs.org](https://nodejs.org) 下载 **LTS 版本**(LTS = 长期支持版,最稳),双击安装,一路下一步。

> LTS 版本要足够新——本系列的工具(OpenSpec)要求 Node **20.19 以上**。nodejs.org 当前的 LTS 通常都满足,装最新 LTS 即可。

**检查**:终端里运行(`-v` 是查版本):

```bash
node -v      # 应显示类似 v22.x.x
npm -v       # 应显示类似 10.x.x
```

两条都显示出版本号,就装好了。显示"command not found"?重启终端再试;还不行就重装 Node。

> 进阶(可跳过):长期用建议装版本管理器(macOS/Linux 用 `nvm` 或 `fnm`,方便切换 Node 版本)。新手先用官网安装包最省事。

## 3. 装 VS Code(代码编辑器)

去 [code.visualstudio.com](https://code.visualstudio.com) 下载安装。这是目前最主流的前端编辑器,免费。

装完建议在 VS Code 里装几个扩展(左侧方块图标 → 搜索 → 安装):

- **ESLint**、**Prettier**(代码检查和格式化)
- 你用的框架相关的(如 Tailwind CSS IntelliSense)

**检查**:能打开 VS Code 即可。

## 4. 装 Git(版本管理)

- **macOS**:终端运行 `git --version`,如果没装会弹窗提示安装"命令行工具",点安装。
- **Windows**:去 [git-scm.com](https://git-scm.com) 下载安装,一路默认。

装完做一次性配置(把名字邮箱换成你的,这会记在你的提交里):

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
```

**检查**:`git --version` 显示版本号即可。

## 5. 装 Claude Code(AI 编程助手)——核心

这是本系列的主角:跑在终端里的 AI 助手。

**安装**(macOS / Linux,终端运行):

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

Windows 用户、或上面的方式不行,可用 npm 装(前面已装 Node):

```bash
npm install -g @anthropic-ai/claude-code
```

**检查安装**:

```bash
claude --version      # 显示版本号(如 2.1.x)
```

**登录**:运行 `claude` 进入,然后输入 `/login`,按提示在浏览器登录。

> 你需要一个 Claude 账号:**Claude.ai 订阅**(Pro/Max,适合个人)或 **API Key**(按量付费)。没有的话先去 [claude.ai](https://claude.ai) 注册。这一步是用 AI 的前提。

**检查**:`claude` 能进入交互界面、`/login` 后显示已登录即可。先 `/exit` 退出,我们待会儿在项目里用它。

> 想系统学 Claude Code 的功能?那是本系列**第一册**的内容。这里只需要会"装好、登录、在项目里启动"。

## 6. 装 OpenSpec(规格驱动工具,推荐)

OpenSpec 让你"先定 spec 再让 AI 写码",避免 AI 跑偏(本册第 6–7 章)。新手也建议装上,后面阶段三会用到:

```bash
npm install -g @fission-ai/openspec
```

**检查**:

```bash
openspec --version    # 显示版本号即可
```

> 装它需要 Node 20.19+(第 2 步)。如果报版本太低,回去升级 Node。

## 阶段一检查清单

全部 ✓ 再进阶段二:

- [ ] 终端能用
- [ ] `node -v` 和 `npm -v` 有版本号(Node ≥ 20.19)
- [ ] VS Code 装好
- [ ] `git --version` 有版本号,且配了 name/email
- [ ] `claude --version` 有版本号,且 `/login` 登录成功
- [ ] `openspec --version` 有版本号

# 阶段二 · 一步步搭建前端项目骨架

我们用 **Vite + React**(2026 新建 SPA 的主流,本册第 1 章),搭一个项目骨架。目标是搭出本文件夹**示例一(任务看板)**的起点。

## 1. 选个地方,建项目

先进到你想放项目的目录(比如桌面或专门的代码文件夹):

```bash
cd ~/Desktop          # macOS 进桌面;Windows 用 cd Desktop
```

用 Vite 脚手架生成项目骨架(一条命令搞定,这就是"脚手架"):

```bash
npm create vite@latest task-board -- --template react-ts
```

- `task-board` 是项目名(也是会生成的文件夹名)。
- `react-ts` 表示用 React + TypeScript 模板。

**检查**:当前目录下多了一个 `task-board` 文件夹。

## 2. 进项目、装依赖

```bash
cd task-board       # 进入项目
npm install         # 下载项目需要的代码库(React 等),会建一个 node_modules 文件夹
```

`npm install` 可能要等一会儿(在下载东西)。

**检查**:出现了 `node_modules` 文件夹,没有红色报错。

## 3. 跑起来,在浏览器看到它

```bash
npm run dev
```

终端会显示一个网址,通常是 `http://localhost:5173`。**按住 Cmd/Ctrl 点它,或手动在浏览器打开。**

**检查**:浏览器里看到 Vite + React 的默认欢迎页面(有个转圈的 logo 和计数按钮)。看到了——恭喜,你的第一个前端项目跑起来了!

> 这个终端窗口现在被开发服务器占用着(它在持续运行)。**别关它。** 要运行别的命令,**新开一个终端窗口**(在项目目录里)。要停掉服务器,在它的窗口按 `Ctrl + C`。

## 4. 用 VS Code 打开,认识目录

新开一个终端,在项目目录里运行:

```bash
code .       # 用 VS Code 打开当前文件夹(. 表示当前目录)
```

(如果 `code` 命令不存在:在 VS Code 里按 `Cmd/Ctrl+Shift+P`,搜 "Shell Command: Install 'code' command",装一下。)

你会看到项目结构,认识几个关键的:

```
task-board/
├── src/              # 你的代码住在这里
│   ├── main.tsx      #   入口文件
│   └── App.tsx       #   主组件
├── index.html        # 页面骨架
├── package.json      # 项目信息 + 依赖 + 命令(scripts)
└── vite.config.ts    # Vite 配置
```

试着改一下 `src/App.tsx` 里的一句文字,保存——浏览器**自动刷新**显示你的改动(这叫热更新,Vite 的快是新手就能感受到的)。

## 5. 把 Claude Code 接进项目

在项目目录里(新终端),启动 Claude Code:

```bash
claude
```

它会以这个项目为工作区。试一句:

```
> 介绍一下这个项目的结构,我是新手
```

它会读项目、给你讲解。这就是 AI 辅助的起点——**它能看你的代码、帮你改、帮你跑命令。**

> 第一册详细讲 Claude Code 怎么用。新手现在只需知道:在项目目录运行 `claude`,然后用自然语言提需求。随时按 `Esc` 打断它,输入 `/exit` 退出。

## 6. 立项目约定(AI 友好的第一步)

这一步让 AI(和将来的你)知道这个项目的约定。本册第 4 章讲原理,新手版很简单——建一个 `AGENTS.md`:

可以直接让 Claude Code 帮你建:

```
> 给这个项目建一个 AGENTS.md,写明:用 Vite + React 19 + TypeScript,
  服务端状态用 TanStack Query,按功能组织目录(src/features/),
  用 React Compiler 不要手动 memo。再建一个 CLAUDE.md 用 @import 引用 AGENTS.md。
```

> 可以参考本文件夹 [示例一的 AGENTS.md](01-Vite-React-任务看板/AGENTS.md) 抄一份。这就是"工具中立的项目约定"(第 4 章)——以后 AI 照它来写,不跑偏、不给过时代码。

## 阶段二检查清单

- [ ] 用 Vite 脚手架建好了项目
- [ ] `npm install` 成功,有 node_modules
- [ ] `npm run dev` 能在浏览器看到页面
- [ ] 用 VS Code 打开了项目,改文字能热更新
- [ ] 在项目里能运行 `claude` 并对话
- [ ] 有了 AGENTS.md(项目约定)

# 阶段三 · 后续如何开发

环境和骨架都有了,现在是日常怎么开发。核心是一个**可重复的循环**。

## 日常开发循环

```
①想做什么(一个功能/一处改动)
   ↓
②告诉 Claude Code(描述目标,不用教它每一步)
   ↓
③它实现 → 你在浏览器看效果(开发服务器自动刷新)
   ↓
④跑测试验证 / 自己看 diff
   ↓
⑤用 Git 保存(commit)
   ↓
回到 ①
```

下面把每一环讲清。

## 用 Claude Code 的基本姿势(新手版)

在项目目录运行 `claude`,然后:

- **给目标,不给步骤**:说"给任务看板加一个清空已完成任务的按钮",而不是"打开 X 文件第 Y 行改成……"。
- **让它验证**:加一句"改完跑一下确认能用"。
- **跑偏就按 `Esc`** 打断,补充说明。别干等它做错。
- **复杂的先让它出方案**:按 `Shift+Tab` 切到 Plan Mode,它会先给计划、你审完再让它动手(第一册第 4 章)。
- **一个任务做完**:输入 `/clear` 清空,开始下一个任务(保持干净)。

## 第一次用 OpenSpec 加一个功能(规格驱动入门)

小改动直接让 AI 做就行。**较大的功能**(比如"加一个任务筛选"),用 OpenSpec 先定 spec,AI 不容易跑偏(本册第 6–7 章)。新手第一次走一遍:

```bash
# 在项目目录(只需做一次)
openspec init
```

然后在 Claude Code 里:

```
/opsx:propose "任务筛选功能"
```

AI 会生成几个文件:**proposal.md**(为什么做)、**specs/**(要建什么、验收标准)、**design.md**(怎么实现)、**tasks.md**(分几步)。

**关键:你先读一遍这些 spec,确认它理解对了。** 比如确认"筛选"是按完成状态分"全部/未完成/已完成"。不对就让它改 spec(改文字,比改代码便宜)。审满意后:

```
/opsx:apply        # AI 照 spec 一步步实现
```

实现完、验证没问题:

```
/opsx:archive      # 归档这次变更
```

> 这就是"规格驱动开发":**先就要建什么达成一致,再让 AI 写。** 体会一下——它比直接说"加个筛选"然后看 AI 自由发挥,确定性高很多。

## 怎么验证"做对了"

前端的好处:能**看到**。两种验证:

1. **在浏览器看**:开发服务器(`npm run dev`)开着,改动自动刷新,你直接点、试。
2. **跑测试**(更可靠):

```bash
npm test           # 跑组件测试(需先配好 Vitest,见示例一)
```

让 Claude Code"改完跑 `npm test` 确认"——它就有了客观的"对没对"依据,不全靠你肉眼(本册第 4 章)。

## 用 Git 保存进度(每完成一点就存)

像游戏存档。每完成一个小功能就保存一次:

```bash
git add .                          # 把所有改动加入暂存
git commit -m "feat: 加任务筛选功能"  # 保存,带一句说明
```

> `feat:` 是约定的前缀(新功能),`fix:` 是修 bug。养成习惯,历史好读(第六册第 5 章)。也可以让 Claude Code 帮你写 commit:"帮我把当前改动提交,写规范的 message"。

第一次在一个新项目用 Git,先初始化一次:

```bash
git init           # 把项目变成 git 仓库(只需一次)
```

(用 Vite 脚手架建的项目,通常已经 init 过了。)

## 遇到问题怎么办(常见报错)

新手最常碰到的:

| 现象 | 多半是 | 怎么办 |
|---|---|---|
| `command not found: node/npm` | Node 没装好或终端没刷新 | 重启终端;重装 Node |
| `command not found: claude` | Claude Code 没装好 | 重看阶段一第 5 步;重启终端 |
| `npm install` 报错 | 网络问题 / Node 版本 | 重试;检查 `node -v` ≥ 20.19 |
| 浏览器打不开 localhost:5173 | 开发服务器没在跑 | 确认那个终端在跑 `npm run dev`,没关 |
| 改了代码浏览器没变 | 服务器没跑 / 保存了吗 | 确认 `npm run dev` 在跑、文件已保存 |
| 一堆红色文字看不懂 | 报错信息 | **直接把它贴给 Claude Code**:`! 上面的命令报错了` 然后让它分析 |

> **最实用的一招**:任何报错,复制下来贴给 Claude Code,问"这个错怎么解决,我是新手"。它能读懂报错并教你修——这正是 AI 辅助的价值。
