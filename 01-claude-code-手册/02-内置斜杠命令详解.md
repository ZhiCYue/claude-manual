# 第 2 章 · 内置斜杠命令详解

会话中输入 `/` 即弹出全部命令的自动补全列表。本章先给全景表，再深挖最值得精通的几个。

## 2.2 深挖：`/clear` —— 最被低估的命令

**用法**：任务完成后输入 `/clear`，对话清零，CLAUDE.md 等持久配置不受影响。

**为什么高手用得最多**：回顾 0.4 节，旧任务的残留（读过的文件、跑过的命令输出）会持续占用窗口并稀释注意力。一个干净窗口里的 Claude，明显比拖着 10 万 token 历史的 Claude 更聪明、更快、更便宜。

**习惯养成**：把 `/clear` 当作"任务的句号"。判断标准——下一件事**不需要**引用这次对话的内容，就 clear。

```
> 修复了登录 bug，测试已通过 ✓
> /clear                          ← 句号
> 接下来实现导出 CSV 功能……        ← 新任务，干净的窗口
```

## 2.3 深挖：`/compact` —— 有损压缩，主动用好过被动等

**用法**：

```
/compact
/compact 重点保留：数据库 schema 的设计决策和未完成的迁移步骤
```

**原理**：harness 让模型把整段历史总结成一份摘要，然后用「摘要 + 最近几条消息」替换原历史。这是**有损**的——细节（具体代码片段、精确报错）会丢失，结论和决策会保留。

**两个实践要点**：

1. **带指示压缩**。你比模型清楚什么必须保留，一句指示能显著提高压缩质量。
2. **主动压缩优于自动压缩**。窗口快满时会自动 compact，但那时机是被动的（可能正卡在任务中间）。在「一个阶段刚结束、下一阶段要开始」的节点主动 compact，丢失最小。

**`/clear` vs `/compact` 决策**：下个任务和这次对话无关 → `clear`；同一个长任务要继续 → `compact`。

## 2.4 深挖：`/context` —— 你的上下文仪表盘

输出类似：

```
Context Usage          claude-fable-5 · 86k/200k tokens (43%)
⛁ System prompt: 3.2k   ⛁ System tools: 12.1k
⛁ MCP tools: 8.4k       ⛁ Memory files: 2.1k
⛁ Messages: 60.2k       ⛀ Free space: 114k (57%)
```

读懂它，你能做出这些判断：

- **MCP tools 占了 2 万多？** → 卸载不用的 MCP 服务器（第 7 章）。
- **Memory files 异常大？** → CLAUDE.md 写太长了，精简（第 3 章）。
- **Messages 增长飞快？** → 该 `/compact` 或把调研工作交给子代理（第 8 章）。

建议入门头两周每天看几次，之后你会形成"窗口体感"，不看也知道该清理了。

## 2.5 深挖：`/rewind` —— 检查点与时光机

**原理**：Claude Code 在每次修改文件前自动创建检查点（独立于 git，存在本地）。`/rewind`（或连按 `Esc Esc`）打开检查点列表，可选择恢复：

- 只恢复**对话**到某个时间点
- 只恢复**代码**到某个时间点
- 两者同时恢复

**典型场景**：让它尝试一个激进方案，跑偏了——`/rewind` 回到尝试前，换个思路重来。这让"大胆实验"的成本趋近于零。

**注意**：检查点不替代 git。它只覆盖 Claude 的文件编辑（不含 `rm`、数据库变更等命令的副作用），且不长期保存。正式节点该 commit 还是要 commit。

## 2.6 深挖：`/model` 与 `/fast` —— 按任务选模型

| 模型 | 特点 | 适合 |
|---|---|---|
| **Fable**（Mythos 级） | 最强推理 | 架构设计、疑难 bug、大型重构 |
| **Opus** | 强且均衡 | 日常复杂开发 |
| **Sonnet** | 快、省 | 常规编码、批量修改 |
| **Haiku** | 最快最省 | 机械性任务、海量小修改 |

```
/model            # 交互式选择
/fast             # Opus 快速输出模式开关
```

会话中途切换模型**不丢上下文**——常见组合拳：用 Fable 做完方案设计，切 Sonnet 执行批量机械修改，省时省额度。

## 2.7 深挖：`/export` 与会话复用

```
/export                  # 导出对话记录
claude --resume          # 跨天恢复长期任务
claude --resume --fork-session   # 从历史会话分叉出新会话（原会话不动）
```

`--fork-session` 适合"同一个调研基础上尝试两个不同方案"：调研一次，分叉两条执行线。
