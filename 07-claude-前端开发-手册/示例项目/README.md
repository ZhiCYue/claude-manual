# 实战项目示例

两个用**当前主流框架**的完整、可跟着做、能跑通的示例项目。它们不是普通的"todo 教程"——而是**演示本册的方法**:在 AI 友好的环境里(支柱一)、用规格驱动的方式(支柱二)、配合 Claude Code 构建真实功能。

> 🔰 **完全的新手?先从这里开始**:[00 · 新手上路:从零搭建](00-新手上路-从零搭建.md)——假设你电脑什么都没装,从装软件(含 Node、VS Code、Git、Claude Code、OpenSpec)、到搭起项目骨架、到日常怎么开发,手把手一步步带。走通它再来做下面两个示例。

## 两个示例覆盖两条主流路线

| 示例 | 框架 | 演示什么 | 何时选这条路 |
|---|---|---|---|
| [01-Vite-React-任务看板](01-Vite-React-任务看板/) | **Vite + React 19 + TS** | SPA 主流栈:TanStack Query 管服务端状态、按功能组织、Vitest + Playwright | 纯客户端 SPA、仪表盘、内部工具(第 1 章) |
| [02-Nextjs-笔记应用](02-Nextjs-笔记应用/) | **Next.js App Router** | 全栈主流栈:Server Components 默认、Server Actions、useActionState | 要 SSR/SEO/全栈一体(第 1 章) |

> 对应第 1 章的选型:新建 **SPA 用 Vite**,**全栈用 Next.js**。webpack 已偏 legacy,不在主流示例里(要用 webpack 见第 2 章)。

## 每个示例怎么用

每个示例文件夹里有:

- **`操作指南.md`** —— 从零搭到跑通的分步指南,且每一步都标注用到本册哪个原则/哪一章。这是核心,先读它。
- **源码文件** —— 真实、一致、可组装成能跑的应用的关键文件(组件、hook、配置、测试)。
- **`AGENTS.md`** —— 该项目的约定(工具中立唯一来源,第 4 章)。

**怎么跟着做**:照 `操作指南.md` 一步步来。指南会告诉你哪些命令自己跑、哪些代码从源码文件复制、哪些步骤交给 Claude Code(用规格驱动)。

## 两个示例共同演示的方法

无论哪个示例,都贯穿本册的两大支柱和完整工作流:

```
① 选对主流框架(第1章)              —— Vite 或 Next
② AI 友好环境(第4章)               —— 按功能组织、AGENTS.md、类型
③ 工具中立质量门(第5章)            —— Vitest/Playwright/lint 进 CI
④ 规格驱动建功能(第6-7章)          —— 较大功能先定 spec 再让 AI 实现
⑤ 地面真相验证(第4章)             —— Playwright 实际操作浏览器
```

读完两个示例,你就有了"主流框架 + AI 辅助 + 规格驱动"的完整实操模板,可以套用到自己的项目。
