# 附录 · 速查表与排错

## A. 快捷键

| 按键 | 作用 |
|---|---|
| `Esc` | 中断 Claude 当前操作（不丢上下文） |
| `Esc Esc` | 回溯历史消息 / 打开检查点回退 |
| `Shift+Tab` | 循环切换权限模式（default → acceptEdits → plan） |
| `↑` / `↓` | 浏览输入历史 |
| `Tab` | 路径自动补全 |
| `Ctrl+C` | 清空当前输入；再按一次退出 |
| `Ctrl+R` | 展开/收起详细输出 |
| `\` + `Enter` | 换行（跑过 `/terminal-setup` 后可用 Shift+Enter） |
| `Cmd+V` | 粘贴图片 |

## B. 输入前缀

| 前缀 | 作用 | 示例 |
|---|---|---|
| `@` | 引用文件/目录进上下文 | `@src/main.py` |
| `!` | 立即执行 shell 命令，输出进对话 | `! git status` |
| `#` | 存入 CLAUDE.md 记忆 | `# 测试用 make test` |
| `/` | 调出斜杠命令 | `/clear` |

## C. CLI 命令速查

```bash
claude                          # 交互式会话
claude "初始指令"                # 带指令启动
claude -c                       # 继续最近会话
claude --resume                 # 选择历史会话（--fork-session 可分叉）
claude -p "任务"                 # 无头模式
claude --model sonnet           # 指定模型
claude --permission-mode plan   # 以 plan 模式启动
claude --add-dir ../lib         # 额外授权目录
claude --effort high            # 思考深度 low/medium/high/xhigh/max
claude mcp add/list/get/remove  # 管理 MCP 服务器
claude doctor                   # 健康自检
claude update                   # 手动更新
```

## E. settings.json 完整注释版

`~/.claude/settings.json`（用户级）或 `项目/.claude/settings.json`（团队级）：

```jsonc
{
  // ── 权限（第 4 章）─────────────────────────
  "permissions": {
    "allow": ["Read(**)", "Bash(git status)", "Bash(npm test *)"],
    "ask":   ["Bash(git push *)"],
    "deny":  ["Read(.env*)", "Bash(rm -rf *)"],
    "additionalDirectories": ["../shared-lib"],   // 默认额外可访问的目录
    "defaultMode": "acceptEdits"                  // 默认权限模式
  },

  // ── 模型与行为 ─────────────────────────────
  "model": "claude-fable-5",          // 默认模型

  // ── 环境变量（每个会话注入）────────────────────
  "env": {
    "NODE_ENV": "development"
  },

  // ── Hooks（第 6 章）────────────────────────
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [{ "type": "command", "command": "..." }]
      }
    ]
  },

  // ── 状态栏（/statusline 可代为生成）────────────
  "statusLine": {
    "type": "command",
    "command": "echo \"$(basename $PWD) | $(git branch --show-current 2>/dev/null)\""
  }
}
```

## H. 常见问题排错（FAQ）

**Q1：Claude 总是忘记/不遵守某条约定？**
按持久度升级：对话里说（最弱）→ 写进 CLAUDE.md（每次会话生效）→ 写成 Hook（程序强制，100%）。反复需要提醒 = 该升级了。另检查 CLAUDE.md 是否太长把重点淹没了。

**Q2：感觉它"变笨了"，答非所问？**
十有八九是上下文太长/太脏。`/context` 确认占用，然后 `/compact`（同任务继续）或 `/clear`（新任务）。MCP tools 占用过大就卸载闲置服务器。

**Q3：权限弹窗太频繁，烦？**
跑 `/fewer-permission-prompts` 自动生成 allowlist；或日常切到 acceptEdits 模式；或手动把高频安全操作加进 `permissions.allow`。**不要**图省事上 bypassPermissions。

**Q4：改坏了怎么回退？**
三层保险：`/rewind`（Claude 的检查点，最快）→ `git checkout/stash`（你的版本控制）→ 检查点不覆盖 shell 命令副作用，所以危险命令前本来就该 commit。

**Q5：Hook 没触发？**
`claude --debug hooks` 看日志；确认 JSON 写在生效层级的 settings 里；settings.json 改动需**新会话**生效；matcher 是工具名的正则不是文件名。

**Q6：MCP 服务器连不上？**
`claude mcp list` 看状态 → `/mcp` 看详情和授权 → stdio 类先手动跑一遍启动命令看报错（如 `npx @playwright/mcp@latest`）→ `claude --debug` 看握手日志。

**Q7：上下文经常爆，长任务做不完？**
组合拳：文档接力法（10.4 节）拆阶段 + 调研交给子代理 + 阶段间主动 `/compact 保留指示`。

**Q8：无头模式在 CI 里卡住不动？**
多半在等权限确认。检查 `--allowedTools` 是否覆盖了所需操作；加 `--max-turns` 设上限；用 `--output-format stream-json --verbose` 看它卡在哪一步。

**Q9：额度/费用烧得快？**
`/cost` 定位大头。常见元凶：从不 `/clear` 的马拉松会话、用 Fable 干 Haiku 的活、无 `--max-turns` 的失控脚本。对照 10.9 节清单逐项治理。

**Q10：怎么让团队新人快速用上这套配置？**
把 `CLAUDE.md`、`.claude/settings.json`、`.claude/commands|agents|skills/`、`.mcp.json` 全部进 git。新人 clone 下来即继承全套环境——这正是"为 AI 设计环境"的资产化价值。
