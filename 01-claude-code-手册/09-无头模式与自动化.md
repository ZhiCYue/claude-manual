# 第 9 章 · 无头模式与自动化

`-p`（`--print`）让 Claude Code 变成一个**可编程的 Unix 命令**：非交互执行一次任务，结果打到 stdout，然后退出。从此它可以出现在管道、脚本、git hook、CI、定时任务里。

## 9.1 基础用法

```bash
# 问完即走
claude -p "用一句话总结这个项目是做什么的"

# 管道：处理 stdin
cat error.log | claude -p "分析这份日志，给出根因和修复建议"
git diff | claude -p "给这个 diff 写一段中文 code review"

# 组合进管道
claude -p "列出本项目所有 TODO，输出纯文本每行一条" | wc -l
```

> **原理**：无头模式跑的是同一个 Agent Loop（会读文件、跑命令、循环修正），区别只是没有人坐在终端前确认权限。所以无人值守的关键，是**用参数预先声明边界**——见 9.3。

## 9.2 输出格式：让程序消费结果

```bash
# 默认 text：只输出最终回答
claude -p "..." --output-format text

# json：附带元数据，适合脚本判断
claude -p "统计 src 各语言文件数" --output-format json
```

json 输出结构（节选关键字段）：

```json
{
  "type": "result",
  "subtype": "success",
  "is_error": false,
  "result": "Python 42 个，TypeScript 18 个……",
  "session_id": "abc-123",
  "total_cost_usd": 0.0142,
  "num_turns": 6,
  "duration_ms": 18234
}
```

脚本里配合 `jq`：

```bash
out=$(claude -p "检查 @config/prod.yaml 是否有安全隐患" --output-format json)
echo "$out" | jq -r '.result'
echo "本次花费：$(echo "$out" | jq '.total_cost_usd') 美元"
```

```bash
# stream-json：流式输出每一步事件（每行一个 JSON）
# 适合在程序里实时监控它"正在干什么"
claude -p "重构 utils 模块" --output-format stream-json --verbose
```

## 9.3 无人值守的安全边界（关键）

```bash
claude -p "修复所有 ESLint 错误" \
  --allowedTools "Read" "Edit" "Bash(npx eslint*)" \
  --disallowedTools "WebFetch" \
  --permission-mode acceptEdits \
  --max-turns 30
```

| 参数 | 作用 |
|---|---|
| `--allowedTools` | 免确认放行的工具白名单（语法同 4.2 节规则） |
| `--disallowedTools` | 黑名单 |
| `--permission-mode` | acceptEdits / plan / bypassPermissions |
| `--max-turns` | 限制循环轮数，防失控 |
| `--fallback-model` | 主模型过载时自动降级（如 `sonnet`），保证脚本不挂 |

原则：**白名单收到刚好够用**。上例中它能读、能改、能跑 eslint，但不能 `git push`、不能联网——出错的爆炸半径被限死了。

会话续接（多步脚本共享上下文）：

```bash
sid=$(claude -p "通读本仓库，建立理解" --output-format json | jq -r '.session_id')
claude -p --resume "$sid" "基于刚才的理解，列出最值得重构的 3 个模块"
claude -p --resume "$sid" "对第 1 个模块给出重构方案"
```

## 9.4 实战一：git pre-commit hook

`.git/hooks/pre-commit`：

```bash
#!/bin/bash
# 提交前让 Claude 快速安检暂存区改动
result=$(git diff --cached | claude -p \
  "审查这个 diff：只找两类问题——硬编码的密钥/密码、明显的逻辑错误。
   没问题就只输出 OK，有问题列出来" \
  --model haiku --output-format text --max-turns 3)

if [[ "$result" != "OK" ]]; then
  echo "⚠️ Claude 安检发现问题："
  echo "$result"
  exit 1   # 阻止提交
fi
```

用 haiku、限 3 轮——快和便宜在这种高频场景比聪明更重要。

## 9.5 实战二：GitHub Actions 自动审 PR

方式一：官方 Action（推荐，在 PR 评论区 @claude 即可互动）：

```yaml
# .github/workflows/claude.yml
name: Claude PR Review
on:
  pull_request:
    types: [opened, synchronize]
jobs:
  review:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      pull-requests: write
    steps:
      - uses: actions/checkout@v4
        with: { fetch-depth: 0 }
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
          prompt: |
            审查本 PR：重点看安全问题、逻辑错误、性能隐患。
            用中文把审查意见发成 PR 评论，按严重程度分级。
```

（更省事的安装方式：会话里跑 `/install-github-app`。）

方式二：裸 CLI（完全自定义）：

```yaml
      - name: Claude review
        run: |
          npm install -g @anthropic-ai/claude-code
          claude -p "审查 git diff origin/main...HEAD 的改动，输出 markdown 报告" \
            --allowedTools "Read" "Grep" "Glob" "Bash(git diff*)" "Bash(git log*)" \
            --output-format text > review.md
          gh pr comment ${{ github.event.number }} --body-file review.md
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          GH_TOKEN: ${{ github.token }}
```

## 9.6 实战三：批量处理脚本

```bash
#!/bin/bash
# 给所有缺少模块级 docstring 的 Python 文件补文档
for f in $(grep -rL '^"""' src --include="*.py"); do
  echo "处理 $f ..."
  claude -p "给 @$f 补充中文 docstring（模块级 + 所有公开函数），
             只加文档，不改任何逻辑" \
    --allowedTools "Read" "Edit" \
    --permission-mode acceptEdits \
    --model sonnet --max-turns 10
done
```

要点：每个文件一次独立调用（隔离失败影响）、只授权 Read/Edit、机械任务用 sonnet。

## 9.8 通往 Agent SDK

`claude -p` 够用是够用，但如果你想**在自己的产品里**内嵌这种代理能力，下一级台阶是 **Claude Agent SDK**（Python/TypeScript）——Claude Code 的引擎以库的形式开放：

```python
# pip install claude-agent-sdk —— 一个最小可运行示例
import anyio
from claude_agent_sdk import query, ClaudeAgentOptions

async def main():
    options = ClaudeAgentOptions(
        allowed_tools=["Read", "Grep", "Glob"],
        max_turns=10,
    )
    async for message in query(prompt="总结这个仓库的架构", options=options):
        print(message)

anyio.run(main)
```

同样的 Agent Loop、工具、权限模型，但宿主是你的程序。这是本系列下一册《Claude API / Agent SDK 开发手册》的主题。
