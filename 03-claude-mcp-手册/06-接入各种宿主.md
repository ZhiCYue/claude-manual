# 第 6 章 · 接入各种宿主

你的服务器写好了，这一章把它接进各种 AI 应用。同一个服务器，处处可用——这是开放协议的核心回报。

---

## 6.1 接入清单一览

| 宿主 | 怎么接 | 传输 |
|---|---|---|
| Claude Code | `claude mcp add` 命令 | stdio / HTTP |
| Claude 桌面端 | `claude_desktop_config.json` | stdio / HTTP |
| Claude API | `mcp_servers` 参数 | HTTP |
| Agent SDK | `mcp_servers` 选项 | stdio / 进程内 |
| VS Code / Cursor 等 | 各自的 MCP 配置 | stdio / HTTP |

## 6.2 Claude Code

最快的方式，命令行注册（第 1 章已讲，这里补全作用域）：

```bash
# 本地 stdio 服务器
claude mcp add weather -- uv --directory /路径/weather run weather.py

# 远程 HTTP 服务器
claude mcp add --transport http weather https://your-server.example.com/mcp

# 带环境变量（如 API key）
claude mcp add --env API_KEY=secret weather -- python /路径/weather.py
```

**作用域**决定配置存哪、谁能用：

```bash
claude mcp add weather ...                    # 默认：仅你、仅本项目
claude mcp add --scope project weather ...     # 生成 .mcp.json，进 git，团队共享
claude mcp add --scope user weather ...        # 你的所有项目可用
```

`--scope project` 生成的 `.mcp.json` 提交进仓库后，团队成员 clone 下来即继承这个服务器配置——这是分享 MCP 服务器给团队的标准做法。

管理：`claude mcp list`、`claude mcp get <名字>`、`claude mcp remove <名字>`；会话内 `/mcp` 查看状态和授权。

## 6.3 Claude 桌面端

通过配置文件（第 1 章已讲完整格式）。本地服务器：

```json
{
  "mcpServers": {
    "weather": {
      "command": "uv",
      "args": ["--directory", "/绝对路径/weather", "run", "weather.py"]
    }
  }
}
```

带环境变量：

```json
{
  "mcpServers": {
    "tickets": {
      "command": "python",
      "args": ["/绝对路径/tickets.py"],
      "env": {
        "TICKETS_API_KEY": "your-key-here"
      }
    }
  }
}
```

改完**重启桌面端**生效。配置位置：macOS `~/Library/Application Support/Claude/claude_desktop_config.json`，Windows `%AppData%\Claude\claude_desktop_config.json`。

## 6.4 Claude API（直连远程服务器）

API 用 `mcp_servers` 参数直连远程 HTTP 服务器（第 5.7 节已示范）：

```python
response = client.beta.messages.create(
    model="claude-opus-4-8",
    max_tokens=4096,
    mcp_servers=[{
        "type": "url",
        "name": "weather",
        "url": "https://your-server.example.com/mcp",
    }],
    betas=["mcp-client-2025-11-20"],
    messages=[{"role": "user", "content": "北京天气？"}],
)
```

注意 API 直连只支持**远程 HTTP** 服务器（API 跑在 Anthropic 那边，连不了你本地的 stdio 进程）。要在 API 里用本地服务器，见下一节的 Agent SDK 或 SDK 的 MCP 转换助手。

### Python SDK 的本地 MCP 助手

第二册提过，`anthropic[mcp]` 提供把本地 MCP 服务器的工具转成 API 工具的助手，配合 Tool Runner 使用：

```python
from anthropic import AsyncAnthropic
from anthropic.lib.tools.mcp import async_mcp_tool
from mcp import ClientSession
from mcp.client.stdio import stdio_client, StdioServerParameters

client = AsyncAnthropic()

async with stdio_client(StdioServerParameters(command="python", args=["weather.py"])) as (read, write):
    async with ClientSession(read, write) as mcp_client:
        await mcp_client.initialize()
        tools = await mcp_client.list_tools()
        runner = client.beta.messages.tool_runner(
            model="claude-opus-4-8",
            max_tokens=4096,
            messages=[{"role": "user", "content": "北京天气？"}],
            tools=[async_mcp_tool(t, mcp_client) for t in tools.tools],
        )
        async for message in runner:
            print(message)
```

这让你在自己的 API 程序里用上本地 MCP 服务器。

## 6.5 Agent SDK

第二册第 8 章的 Agent SDK 通过 `mcp_servers` 选项接入。它既能连外部 MCP 服务器，也能用**进程内**的 SDK MCP 服务器（`create_sdk_mcp_server`，第二册 8.5 节）：

```python
from claude_agent_sdk import ClaudeAgentOptions

options = ClaudeAgentOptions(
    mcp_servers={
        # 外部 stdio 服务器
        "weather": {"command": "python", "args": ["weather.py"]},
        # 或进程内服务器（见第二册 8.5）
        "calc": calculator_server,
    },
    allowed_tools=["mcp__weather__get_forecast", "mcp__calc__add"],
)
```

工具名格式 `mcp__<服务器名>__<工具名>` 在所有宿主里一致。

## 6.6 其他宿主

MCP 是开放协议，越来越多 AI 应用支持它：

- **VS Code**：通过其 MCP 扩展配置。
- **Cursor**：设置里的 MCP 配置。
- **其他 IDE / Agent 框架**：各有各的配置入口，但格式大同小异——指定启动命令（stdio）或 URL（HTTP）。

无论哪个宿主，你需要提供的信息就两类：

- **本地服务器**：启动命令（`command` + `args`）+ 可选环境变量。
- **远程服务器**：URL + 可选鉴权。

> **这就是"写一次处处可用"的实感**：你为天气服务器写的同一份 `weather.py`，加进 Claude Code 的命令、桌面端的 JSON、API 的 `mcp_servers`、VS Code 的配置——业务代码一行没改。这是 MCP 作为标准协议的全部意义。

## 6.7 上下文成本提醒

呼应第一册第 7.7 节：**每个连接的 MCP 服务器，其工具/资源/提示词的声明都常驻宿主的上下文窗口**。装太多服务器会吃掉大量 token 并稀释模型注意力。

写服务器时这意味着：

- **工具集聚焦**——别在一个服务器里塞几十个工具。
- **描述精炼但清晰**——描述要够模型理解，但别啰嗦(它常驻上下文)。
- 用户侧可以用 `/context`（Claude Code）看 MCP tools 占用，决定是否卸载。
